# 题目描述

```TXT
Table: Failed

+--------------+---------+
| Column Name  | Type    |
+--------------+---------+
| fail_date    | date    |
+--------------+---------+
该表主键为 fail_date。
该表包含失败任务的天数.
Table: Succeeded

+--------------+---------+
| Column Name  | Type    |
+--------------+---------+
| success_date | date    |
+--------------+---------+
该表主键为 success_date。
该表包含成功任务的天数.
 

系统 每天 运行一个任务。每个任务都独立于先前的任务。任务的状态可以是失败或是成功。

编写一个 SQL 查询 2019-01-01 到 2019-12-31 期间任务连续同状态 period_state 的起止日期（start_date 和 end_date）。即如果任务失败了，就是失败状态的起止日期，如果任务成功了，就是成功状态的起止日期。

最后结果按照起始日期 start_date 排序

查询结果样例如下所示:

Failed table:
+-------------------+
| fail_date         |
+-------------------+
| 2018-12-28        |
| 2018-12-29        |
| 2019-01-04        |
| 2019-01-05        |
+-------------------+

Succeeded table:
+-------------------+
| success_date      |
+-------------------+
| 2018-12-30        |
| 2018-12-31        |
| 2019-01-01        |
| 2019-01-02        |
| 2019-01-03        |
| 2019-01-06        |
+-------------------+


Result table:
+--------------+--------------+--------------+
| period_state | start_date   | end_date     |
+--------------+--------------+--------------+
| succeeded    | 2019-01-01   | 2019-01-03   |
| failed       | 2019-01-04   | 2019-01-05   |
| succeeded    | 2019-01-06   | 2019-01-06   |
+--------------+--------------+--------------+

结果忽略了 2018 年的记录，因为我们只关心从 2019-01-01 到 2019-12-31 的记录
从 2019-01-01 到 2019-01-03 所有任务成功，系统状态为 "succeeded"。
从 2019-01-04 到 2019-01-05 所有任务失败，系统状态为 "failed"。
从 2019-01-06 到 2019-01-06 所有任务成功，系统状态为 "succeeded"。
```

# 解题

> 典型的连续区间问题，但是需要注意的是它把`连续1天`也包括在内。

```SQL

with base as (
    select
        *
    from (
        select
            'succeeded' as period_state,
            success_date as date
        from
            Succeeded
        union all
        select
            'failed' as period_state,
            fail_date as date
        from
            Failed
    ) as a
    where
        date between '2019-01-01' and '2019-12-31'

)
select
    period_state,
    min(date) as start_date,
    max(date) as end_date
from (
    select
        *,
        count(*) over (partition by period_state, next_date) as num -- 分组统计
    from (
        select
            *,
            date_sub(date, interval ranking day) as next_date -- 计算间隔后的日期
        from (
            select
                *,
                row_number() over (partition by period_state order by date) as ranking -- 排序
            from
                base
        ) as a
    ) as a
) as a
where
    num >= 1
group by
    period_state,
    next_date, -- 这里需要注意，他是保证连续1天的关键
    num  
order by
    start_date

```
